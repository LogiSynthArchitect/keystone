EDGE CASES (from Architect Review)

✖ OOM death spiral: loading all jobs for the month → FIXED with rollups
✖ Margin drift: inventory cost changes retroactively affect GP → FIXED (uses historical unitPrice)
✖ No cached snapshots: re-scan every screen open → FIXED (rollup box persists)
⚠ New customer detection on single-date recompute (scans ALL customer jobs) → acceptable
⚠ Empty date without explicit rollup → FIXED (writes clean daily rollup)
⚠ Inventory cost changes don't invalidate → by design (immutable GP)
