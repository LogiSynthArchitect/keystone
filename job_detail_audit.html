!
MEDIUM
U1. No service-type icon in header
Job card has a 40×40 icon box with per-service icons (via _inferIcon()). Detail screen just shows text. Breaks visual continuity between list and detail views.
!
MEDIUM
U2. Container borderRadius: 4 vs design standard 12
All detail containers use BorderRadius.circular(4). Job card uses 12 (matches AGENTS.md card pattern). Inconsistent visual language.

LOW

⚠️ LOGIC ISSUES

!
HIGH
L1. No status transition validation
_showStatusSheet allows selecting ANY status regardless of rules. Can skip from 'quoted' → 'invoiced' or go backward. JobEntity.validateStatusTransition() exists but is never called.
!
HIGH
L2. No payment transition validation
_showPaymentSheet allows any payment status. Can mark 'paid' on a 'quoted' job, or revert 'paid' → 'unpaid'. JobEntity.validatePaymentTransition() exists but unused.
!
LOW
L3. KsConfirmDialog.onConfirm is dead code
All 4 usages pass onConfirm: () {}. The callback is never reached because actual action checks Future<bool?> return value instead. Misleading API usage.
→ lines 76, 881, 1111, 1212
!
LOW
L4. _buildExpensesSection unnecessarily watches parts
Line 663: watches jobPartsProvider solely to compute net profit. Creates the coupling problem in B2.
