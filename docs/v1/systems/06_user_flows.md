# DOCUMENT 06 — CORE USER FLOW
### Project: Keystone
**Required Inputs:** Document 04 — Core Scope Definition, Document 05 — User Personas
**Location:** Ghana, West Africa
**Date:** 2026
**Status:** APPROVED

---

## Flow 1 — Onboarding (The Identity Setup)

1. **Identity Initiation:** User opens app and enters phone number.
2. **Security Tunnel:** 6-digit OTP verification.
3. **Profile Calibration:** User enters full name and selects service specializations.
4. **Result:** User lands on the Tactical Dashboard.

---

## Flow 2 — Log a Job (The Tactical Wizard)

This flow utilizes the **3-Step Wizard** to maximize field efficiency.

### Step 01: SERVICE
- User selects the service performed (e.g., CAR KEY).
- NEXT is enabled only after selection.

### Step 02: ENTITY
- User enters Customer Name and Phone.
- Integrated search allows selection of existing entities to bypass typing.

### Step 03: LOGISTICS
- User enters Amount (GHS), Location, and Job Notes.
- Integrated field hints provide real-time guidance.
- ACTION: **SAVE JOB RECORD**.

---

## Flow 3 — WhatsApp Professionalism

1. **Drafting:** Immediately after saving, the user is presented with the **Editable Message Preview**.
2. **Personalization:** User can rewrite any part of the "Thank You" message inside Keystone.
3. **Tunneling:** User taps **SEND WHATSAPP**; wa.me link launches WhatsApp with pre-filled content.

---

## Flow 4 — Admin Job Correction

1. **Trigger:** Technician discovers an error on a locked job (>24h).
2. **Request:** Tech taps **REQUEST CORRECTION**, enters reason, and submits.
3. **Review:** Admin logs in, goes to **Admin Dashboard**, and reviews pending requests.
4. **Resolution:** Admin taps **APPROVE**; job is updated and tech is notified in-app.

---

## Flow 5 — Knowledge Retrieval

1. **Log:** Tech saves technical findings via the **2-Step Add Note Wizard**.
2. **Index:** Tech applies category and custom tags.
3. **Search:** Later, tech uses the **Filter Chips** (e.g., [INSTALL]) to find relevant technical docs instantly.
