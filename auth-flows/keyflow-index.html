KEYSTONE

Auth Flow Map — 7 complete user journeys from entry to dashboard
Flow 01

New User

First-time user signs up. Full onboarding: phone, OTP, create password, mandatory PIN, biometric, name, services, terms.

LANDING→PHONE→OTP→CREATE PW→PIN SETUP→BIOMETRIC→ONBOARDING→DASH
Flow 02

Returning User

Existing user signs in with password. Quick unlock path.

LANDING→PHONE→PW ENTRY→TRANSITION→DASH
Flow 03

Upgrade User

Legacy user forced to create password, then mandatory PIN + optional biometric unlock.

LANDING→PHONE→PW ENTRY→UPGRADE→PIN SETUP→BIOMETRIC→DASH
Flow 04

Forgot Access

User forgot password. Recovery flow: request code, reset password.

PHONE→FORGOT→RESET PW→TRANSITION→DASH
Flow 05

Re-auth (Locked)

Device locked due to inactivity / failed PIN attempts. Unlock methods.

LOCKED→PIN / PW→TRANSITION→DASH +STALE DATA
Flow 06

Other Gates

Post-auth checkpoints: terms re-accept, initial sync, version block.

TERMS→SYNC→VERSION→DASH
