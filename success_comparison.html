DECISION

KsSuccessMoment is a celebration ritual — it's meant to feel rewarding when you complete an action. It blocks the screen momentarily so the user can bask in the success before continuing. The pricing screen uses it after a price save because changing a price is a significant action.

KsSlidingNotification is a notification — it informs without interrupting. It's meant for routine feedback (save succeeded, item added, etc.) where the user should continue working.

Options: Replace KsSuccessMoment with KsSlidingNotification (consistent but less celebratory), or keep KsSuccessMoment as a distinct pattern for significant actions.
