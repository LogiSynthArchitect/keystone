🔧 Fix: Use bottomLabel instead

  • Set bottomLabel: "UPDATE PAYMENT" — The built-in bottom bar renders edge-to-edge, gold, with arrow icon. Same as every other drawer in the app.
  • Set canPop: () => false — Prevents automatic pop after onDone. We pop manually inside the async flow.
  • Use onDone to kick off async work — The button fires onDone synchronously, then checks canPop. Since canPop returns false, sheet stays open. The async work (confirm dialog → API call → pop) completes and calls Navigator.pop(ctx) manually.
  • Status/method selection stays inside contentBuilder — No change needed there.

Challenge: onDone is VoidCallback (synchronous).
The async flow (confirm dialog → API call) can't be awaited inside it.

Solution: Start the async flow as fire-and-forget from onDone.
The sheet stays open (canPop returns false). The async flow completes,
then calls Navigator.pop(ctx) + shows notification.
